<p><b>Matthew Dillon</b> (born 1966) is an American <a href="/wiki/Software_engineer" title="Software engineer">software engineer</a> known for <a href="/wiki/Amiga" title="Amiga">Amiga</a> software,<sup id="cite_ref-informit-2007_3-1" class="reference"><a href="#cite_note-informit-2007-3">&#91;3&#93;</a></sup> contributions to <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> and for starting and leading the <a href="/wiki/DragonFly_BSD" title="DragonFly BSD">DragonFly BSD</a> project since 2003.<sup id="cite_ref-informit-2007_3-2" class="reference"><a href="#cite_note-informit-2007-3">&#91;3&#93;</a></sup><sup id="cite_ref-kerneltrap-2002_5-0" class="reference"><a href="#cite_note-kerneltrap-2002-5">&#91;5&#93;</a></sup><sup id="cite_ref-kerneltrap-2007_6-0" class="reference"><a href="#cite_note-kerneltrap-2007-6">&#91;6&#93;</a></sup><sup id="cite_ref-team_7-0" class="reference"><a href="#cite_note-team-7">&#91;7&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Biography">Biography</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Matthew_Dillon&amp;action=edit&amp;section=1" title="Edit section: Biography">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Dillon studied <a href="/wiki/Electronic_engineering" title="Electronic engineering">electronic engineering</a> and <a href="/wiki/Computer_science" title="Computer science">computer science</a> at the <a href="/wiki/University_of_California,_Berkeley" title="University of California, Berkeley">University of California, Berkeley</a>, where he first became involved with <a href="/wiki/BSD" class="mw-redirect" title="BSD">BSD</a> in 1985. He also became known for his <a href="/wiki/Amiga" title="Amiga">Amiga</a> programming,<sup id="cite_ref-informit-2007_3-3" class="reference"><a href="#cite_note-informit-2007-3">&#91;3&#93;</a></sup> his C compiler DICE<sup id="cite_ref-dice-mia_2-1" class="reference"><a href="#cite_note-dice-mia-2">&#91;2&#93;</a></sup> and his work on the <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a>.<sup id="cite_ref-login04_8-0" class="reference"><a href="#cite_note-login04-8">&#91;8&#93;</a></sup> He founded and worked at <a href="/wiki/Verio" title="Verio">Best Internet</a> from 1994 until 1997, contributing to <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> in that time.<sup id="cite_ref-sage-1998_9-0" class="reference"><a href="#cite_note-sage-1998-9">&#91;9&#93;</a></sup> His "Diablo" internet news transit program was very popular with many ISPs.
</p><p>In 1997, Dillon gained <a href="/wiki/Commit_(data_management)" title="Commit (data management)">commit</a> access to the FreeBSD code and heavily contributed to the <a href="/wiki/Virtual_memory" title="Virtual memory">virtual memory</a> subsystem,<sup id="cite_ref-onlamp-2004_10-0" class="reference"><a href="#cite_note-onlamp-2004-10">&#91;10&#93;</a></sup> amongst other contributions.
</p><p>Concerned with problems he saw in the direction FreeBSD 5.x was headed in regards to <a href="/wiki/Concurrency_(computer_science)" title="Concurrency (computer science)">concurrency</a>,<sup id="cite_ref-onlamp-2004_10-1" class="reference"><a href="#cite_note-onlamp-2004-10">&#91;10&#93;</a></sup> and coupled with the fact that Dillon's access to the FreeBSD source code repository was revoked due to a falling-out with other FreeBSD developers, he started the <a href="/wiki/DragonFly_BSD" title="DragonFly BSD">DragonFly BSD</a> project in 2003, implementing the <a href="/wiki/Symmetric_multiprocessing" title="Symmetric multiprocessing">SMP</a> model using <a href="/wiki/Light_Weight_Kernel_Threads" title="Light Weight Kernel Threads">light-weight kernel threads</a>.<sup id="cite_ref-informit-2007_3-4" class="reference"><a href="#cite_note-informit-2007-3">&#91;3&#93;</a></sup><sup id="cite_ref-go_11-0" class="reference"><a href="#cite_note-go-11">&#91;11&#93;</a></sup> The DragonFly project also led to the development of a new <a href="/wiki/Userspace" class="mw-redirect" title="Userspace">userspace</a> kernel <a href="/wiki/OS-level_virtualisation" class="mw-redirect" title="OS-level virtualisation">virtualisation</a> technique in 2006, called <a href="/wiki/Virtual_Kernel" class="mw-redirect" title="Virtual Kernel">Virtual Kernel</a>,<sup id="cite_ref-informit-2007_3-5" class="reference"><a href="#cite_note-informit-2007-3">&#91;3&#93;</a></sup><sup id="cite_ref-lwn-2010_12-0" class="reference"><a href="#cite_note-lwn-2010-12">&#91;12&#93;</a></sup> originally to ease the development and testing of subsequent kernel-level features;<sup id="cite_ref-Reed2007_13-0" class="reference"><a href="#cite_note-Reed2007-13">&#91;13&#93;</a></sup> a new <a href="/wiki/File_system" title="File system">file system</a>, called <a href="/wiki/HAMMER_(file_system)" title="HAMMER (file system)">HAMMER</a>, which he created using <a href="/wiki/B-tree" title="B-tree">B-trees</a>; HAMMER was declared production-ready with DragonFly 2.2 in 2009;<sup id="cite_ref-lwn-2010_12-1" class="reference"><a href="#cite_note-lwn-2010-12">&#91;12&#93;</a></sup> and, subsequently, <a href="/wiki/HAMMER2" title="HAMMER2">HAMMER2</a>, declared stable in 2018 with DragonFly 5.2.
</p><p>Most recently, Dillon has gotten a number of headlines around <a href="/wiki/CPU" class="mw-redirect" title="CPU">CPU</a> <a href="/wiki/Errata" class="mw-redirect" title="Errata">errata</a>.  In 2007, this was after <a href="/wiki/Theo_de_Raadt" title="Theo de Raadt">Theo de Raadt</a> of <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a> raised the alarm around the seriousness of some of the errata for <a href="/wiki/Intel_Core_2" title="Intel Core 2">Intel Core 2</a> family of CPUs.<sup id="cite_ref-lor-2007_14-0" class="reference"><a href="#cite_note-lor-2007-14">&#91;14&#93;</a></sup>  Dillon has independently evaluated Intel's errata, and did an overview of <a href="/wiki/Intel_Core" title="Intel Core">Intel Core</a> errata as well, suggesting that several of the erratums were so serious as to warrant avoiding any processor where the issues remain unfixed.<sup id="cite_ref-lor-2007_14-1" class="reference"><a href="#cite_note-lor-2007-14">&#91;14&#93;</a></sup>  Dillon has since been a fan of <a href="/wiki/AMD" class="mw-redirect" title="AMD">AMD</a> processors, and, subsequently in 2012, he has discovered a brand-new deficiency in some <a href="/wiki/AMD" class="mw-redirect" title="AMD">AMD</a> processors for which no existing erratum existed at the time.<sup id="cite_ref-theregister-amd-2012_15-0" class="reference"><a href="#cite_note-theregister-amd-2012-15">&#91;15&#93;</a></sup>  Dillon continued his work around CPU issues as late as 2018, presenting solutions to tackle the latest security vulnerabilities like <a href="/wiki/Meltdown_(security_vulnerability)" title="Meltdown (security vulnerability)">meltdown</a>, some of which have been subsequently adopted by <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a> as well.<sup id="cite_ref-theregister-openbsd-meltdown-2018_16-0" class="reference"><a href="#cite_note-theregister-openbsd-meltdown-2018-16">&#91;16&#93;</a></sup>
</p><p>Dillon was a frequent guest on bsdtalk during the runtime of the show,<sup id="cite_ref-bsdtalk_17-0" class="reference"><a href="#cite_note-bsdtalk-17">&#91;17&#93;</a></sup> and was interviewed several times for <a href="/wiki/KernelTrap" title="KernelTrap">KernelTrap</a>.<sup id="cite_ref-kerneltrap-2002_5-1" class="reference"><a href="#cite_note-kerneltrap-2002-5">&#91;5&#93;</a></sup><sup id="cite_ref-kerneltrap-2007_6-1" class="reference"><a href="#cite_note-kerneltrap-2007-6">&#91;6&#93;</a></sup>
</p>
